# 034: グラフ表示Web UI

## 状態

未着手。Web UIを公開する方針を決めた後に開始する。描画ライブラリとビルド方法は、
実装前に比較して決定する。

## 目的

要件定義「グラフ可視化」節を実装する。ログイン後の主画面に、利用者が閲覧できる全ノートの
有向グラフを表示し、検索・絞り込み・ノート選択の起点とする。Issue 006（編集プレビュー）は
単一ノートの編集体験を扱い、本 issue は空間全体の可視化を扱う。

## 決定済みの要件

- 要件定義の該当節を正とする。特に次を変更しない。
  - 弱連結成分で最大成分を中央、その他と孤立ノードを周辺へ配置する。
  - リンク切れは警告色の仮ノード、権限のない参照先は匿名の「権限なし」仮ノードとして表示
    し、いずれも最大成分の判定から除外する。
  - 閲覧できないノートのタイトル・タグ・作成者を表示しない。閲覧できないノートから閲覧
    できるノートへの参照は表示しない。root は全ノートを通常ノードとして表示できる。
  - 自己ループは表示しない。同一ノード対の複数参照は一本のエッジへ集約する。
  - レイアウトはクライアント側で計算し、永続化しない。レイアウト情報を公開 API 仕様に
    含めない。
- グラフ取得 API はメタデータとエッジだけを返し、本文はノート選択時に別途取得する。
  初期想定規模（約 1,500 ノート）では一括取得とする。
- フロントエンドは TypeScript とし、同一オリジンから Cookie セッションで API を利用する。
  任意オリジンへの CORS は無効のままとする。

## 実装前に決める事項

1. **グラフ描画基盤**: 力学レイアウトを持つライブラリ（例: Cytoscape.js、Sigma.js、
   D3-force、WebGL 系）を、1,500 ノード規模の描画性能、TypeScript 型、ライセンス、保守状況で
   比較する。
2. **ビルドとパッケージング**: フロントエンドのビルド成果物を Nix ビルドと NixOS モジュールへ
   組み込む方式（ビルド再現性、`nix develop` での開発ループ）を決める。
3. **CSP との整合**: 現在の `default-src 'none'` 系の厳格な CSP を、インラインスクリプトなしで
   成立させる配信方式（ハッシュ付与または外部ファイル化）を決める。
4. **状態と URL**: 選択中ノート ID の URL 反映、ブラウザー履歴、検索・タグ絞り込み状態の扱いを
   決める。

## 作業内容

1. グラフ用の読み取り API（可視ノートのメタデータとエッジの一括取得）を、既存の
   `NoteUseCases` と同じ ACL 非漏洩規則で追加する。OpenAPI へ後方互換に追記する。
2. 弱連結成分の計算と初期配置、ズーム・パン・ドラッグ・選択、検索結果の強調、タグ絞り込みを
   実装する。
3. リンク切れ・権限なし仮ノードの表示と、選択ノートの本文表示（閲覧 UI）を接続する。
4. レイアウト計算中も検索・選択の操作を停止させないことを、想定規模のテストデータで検証
   する。

## 完了条件

- 約 1,500 ノートのフィクスチャで、初期表示・操作が実用的な応答性を保つ。
- ACL 非漏洩規則（タイトル・存在の秘匿、仮ノードの匿名性）がフロントエンドを含めて成立し、
  E2E で検証される。
- グラフ UI の追加が、ノート保存形式・REST 仕様・MCP 仕様へ破壊的変更を持ち込まない。
