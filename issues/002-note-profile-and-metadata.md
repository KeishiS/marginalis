# 002: ノート用AsciiDocプロファイルと属性検証

## 目的

本アプリが保存するノートの文書ヘッダと本文を、AdocWeaveの標準解析後に一貫して検証する。

## 範囲

- `note-id`、`creator-id`、`created-at`、`updated-at`および`tags`を必須属性として検証する。
- UUIDv7、固定ミリ秒精度のUTC RFC 3339日時、タグの正規化・重複除去・上限を検証する。
- `note-id`、`creator-id`および`created-at`を、作成済みノートの保護属性として扱う。
- 明示アンカーと見出しIDを抽出し、重複アンカーを診断する。
- 保存時のstrict modeと、編集途中のpermissive modeの診断severityを定義する。
- 属性名・値の正確なUTF-8 byte rangeを利用して、安定したホスト診断を返す。

## 範囲外

- ACL、現在の管理者およびリビジョンをAsciiDocへ保存すること。
- ノート参照の実在確認。

## 完了条件

- 正常なノートヘッダ、各必須属性の欠落・形式不正・変更禁止違反をfixtureで検証する。
- 属性の原文はFormatterで破壊されず、正規化値だけを投影へ使用する。
- サーバ側の保存検証とブラウザ側の編集中診断が同じプロファイル規則を用いる。
