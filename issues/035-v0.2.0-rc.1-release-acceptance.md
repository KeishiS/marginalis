# 035: v0.2.0-rc.1のリリース受入

## 状態

完了（2026-07-24）。AdocWeave v0.6.1への実装移行、版更新、旧`dataDir`の削除、
実環境での受入確認、受入中の修正を含む最終自動検証、タグ公開を完了した。
GitHub Releaseは作成せず、注釈付きタグを公開物とした。

## 目的

`v0.2.0-rc.1`を、AdocWeave v0.6.1を使う最初の候補版として受け入れるか判定する。
旧データを持ち込まずに空の環境を初期化し、実Kanidm、リバースプロキシー、
実MCPクライアント、実バックアップ保存先で主要経路を確認する。

## 公開範囲

- アプリケーション版を`0.2.0-rc.1`へ更新する。
- AdocWeaveのRust版とWASM版をv0.6.1の同一コミットへ固定する。
- `/api/v1` の OpenAPI 仕様と既存 MCP ツールの仕様は変更しない。
- データフォーマットの識別子はv1を維持するが、その内容はAdocWeave v0.6.1前提で
  破壊的に再定義する。
- 旧v1の移行、復元、切戻しは提供しない。

## 自動判定

候補コミットとタグ作成直前のコミットで、Linux環境から次を実行する。

```bash
nix develop --command cargo make release-gate
```

この検査には、Rustの書式・型検査・Clippy・全テスト、依存境界、OpenAPI、
AdocWeaveの版と供給経路、flake評価、Nixパッケージ、NixOSモジュールVM、
実バイナリVMを含める。タグをpushした後は、GitHub Actionsの`Release gate`も
成功しなければならない。

## 破壊的初期化

旧v1と新しいv1は`FORMAT`マーカーでは区別できない。新しいバイナリを起動する前に、
運用者が次を実施する。

1. サービスを停止し、設定中の`dataDir`の絶対パスを確認する。
2. 必要なら旧`dataDir`を、更新後の復元には使わないオフライン退避物として保存する。
3. 対象パスを再確認して旧`dataDir`全体を削除する。
4. 候補版へ更新し、空の`dataDir`から`root`資格情報と登録ポリシーを初期化する。
5. 旧バックアップ、SQLite、AsciiDoc正本、投影を候補版へ戻さない。

削除はアプリケーションやNixOSモジュールへ委譲しない。実施日時、対象パス、
成否だけを記録し、資格情報やデータ内容は記録しない。

## 手動受入

[実環境での受入確認](../docs/acceptance.md)に従い、次を確認する。

- 空の`dataDir`から起動し、`root`を初期化できる。
- OIDCセッションを使うREST APIの作成、取得、更新、削除、検索。
- 保護属性がサーバー値へ置換され、更新時に不変属性を変更できない。
- ETagによる競合検出と、ACLで不可視なノートの非漏洩。
- `xref:note:`、LaTeX数式、許可したソースコード言語を含む正本を保存できる。
- include、passthrough、危険なURL、未許可のソースコード言語を拒否する。
- 実MCPクライアントのOAuth認可、RESTとMCPの可視範囲の一致、認可取消後の失効。
- 新しいv1で作成したバックアップの検証、非破壊の復元候補、投影再構築。
- 監査タイマーと、稼働中バイナリの OpenAPI 仕様。

記録には成否、日時、候補版、`X-Request-Id`だけを含める。パスワード、Cookie、
OIDCコード、アクセストークン、リフレッシュトークン、クライアントシークレット、
ノート本文は記録しない。

## リリースを妨げる問題

- 旧`dataDir`を削除した空の環境から起動・初期化できない。
- AdocWeaveのRust版、WASM版、投影、配布物のパッケージ版が一致しない。
- 正本、SQLite投影、新しいv1のバックアップ・復元候補の整合性が失われる。
- ACLで不可視なノート、秘密情報、認証情報が漏れる。
- OIDC、MCP OAuth、トークン失効、NixOSモジュールが動作しない。
- 実装が公開 OpenAPI 仕様と一致しない。

Web UI、SMTP、招待、利用者の再有効化、グループACL、管理専用オリジン、mTLS、
PostgreSQLは、この候補版を妨げる問題に含めない。

  `0.2.0-rc.1`で一致する。
