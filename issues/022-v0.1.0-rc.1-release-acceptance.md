# 022: v0.1.0-rc.2のリリース受入判定

状態: 完了（2026-07-23）。手動受入、自動判定、タグ作成を完了した。
GitHub Releaseは作成しない方針とした。

## 目的

`v0.1.0-rc.2`を、研究室内でREST APIとOAuth保護MCPを運用できる候補版として
受け入れるか判定する。自動試験では確認できない実Kanidm、リバースプロキシー、
実MCPクライアント、実バックアップ保存先を確認する。

## 自動判定

タグを作成する前に、Linux環境で次のコマンドが成功することを確認する。

```bash
nix develop --command cargo make release-gate
```

タグをリモートリポジトリへ送信した後は、GitHub Actionsの`Release gate`
ワークフローも成功しなければならない。

## 手動受入

[実環境での受入確認](../docs/acceptance.md)の段階1から3を実施し、次の項目を確認する。

- OIDCセッションを使うREST APIの作成、取得、更新、削除、検索
- ETagによる競合検出
- ACLで不可視なノートの存在を漏らさないこと
- ノートの物理削除
- 実MCPクライアントのOAuth認可
- REST APIとMCPで可視範囲が一致すること
- 認可取消後にトークンが失効すること
- バックアップ作成
- 非破壊の復元候補作成
- 投影の再構築
- 監査タイマー
- 稼働中のOpenAPI契約

記録には、成否、日時、サービスのバージョン、`X-Request-Id`だけを含める。
パスワード、Cookie、OIDCコード、アクセストークン、リフレッシュトークン、
クライアントシークレットは記録しない。

## リリースを妨げる問題

- ACLで不可視なノートの存在が漏れる。
- 正本、SQLite投影、バックアップ、復元の整合性が失われる。
- OIDCコールバック、セッション、MCP OAuth、トークン失効の安全性が損なわれる。
- NixOSモジュールを起動できない。
- 秘密情報が露出する。
- 実装が公開OpenAPI契約と一致しない。

Web UI、SMTP、招待、利用者の再有効化、グループACL、管理専用オリジン、mTLSは、
この候補版を妨げる問題には含めない。

## 完了条件

- 自動判定と手動受入がすべて成功する。
- OpenAPI契約は、リリースを妨げる問題の修正を除いて変更されていない。
- `LICENSE-MIT`、`LICENSE-APACHE`、`CHANGELOG.md`、`docs/release.md`が、
  候補版と既知の制約を正しく記録している。
- 注釈付きタグ`v0.1.0-rc.2`を作成する。
- GitHub Releaseのプレリリースは、公開配布が必要な場合にだけ作成する。
