# 022: v0.1.0-rc.1 release acceptance

状態: 手動受入完了（2026-07-23）。tag作成・GitHub Release待ち。

## 目的

`v0.1.0-rc.1`を、研究室内でREST APIとOAuth保護MCPを実運用できる候補版として判定する。
自動gateでは扱えない実Kanidm、reverse proxy、実MCP clientおよび実backup storageを受入する。

## 自動gate

tagの作成前に、Linux環境で`nix develop --command cargo make release-gate`を成功させる。
tag push後はGitHub Actionsの`Release gate` workflowも成功しなければならない。

## 手動受入

[実環境受入確認](../docs/acceptance.md)の段階1から3を完了し、次を記録する。

- OIDC sessionを使うREST CRUD、検索、ETag競合、ACL非漏洩、物理削除。
- 実MCP clientのOAuth認可、REST/MCPの可視性一致、認可取消後のtoken失効。
- backup generation、非破壊restore候補、投影再構築、監査timer、稼働中OpenAPI contract。

記録は成功・失敗、日時、service version、`X-Request-Id`だけとし、password、Cookie、OIDC code、
access token、refresh tokenおよびclient secretを含めない。

### 実施記録

- 2026-07-23: 段階1から3を完了。REST受入確認と、ChatGPT Web UIを実MCP clientとして用いたOAuth接続・
  連携確認を完了した。secretおよびtoken値は記録しない。

## release blocker

- ACLによる不可視ノートの存在漏洩。
- 正本・SQLite投影・backup/restoreの整合性喪失。
- OIDC callback、session、MCP OAuthまたはtoken失効のsecurity上の破綻。
- NixOS moduleの起動不能、secret露出、または公開OpenAPI contractとの差異。

Web UI、SMTP、招待、ユーザー再有効化、グループACL、専用管理origin・mTLSはRC.1のblockerではない。

## 完了条件

- 自動gateと手動受入が全て成功する。
- OpenAPI contractはrelease blocker修正以外で変更されていない。
- `LICENSE-MIT`、`LICENSE-APACHE`、`CHANGELOG.md`および`docs/release.md`が候補版と既知の制約を正しく記録する。
- annotated tag `v0.1.0-rc.1`を作成し、GitHub Releaseをprereleaseとして発行する。
